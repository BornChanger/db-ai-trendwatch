# Source Map

Prioritize primary sources first. Use secondary sources only to find pointers.

## Primary sources
- SaaS vendor engineering blogs and architecture posts
- Incident reports and postmortems
- Release notes and changelogs
- Public architecture talks (QCon, SREcon, VLDB/SIGMOD industry tracks)
- Open-source repos and release tags for self-hosted products

## Secondary sources
- Cloud provider blogs and reference architectures
- Database vendor case studies
- Analyst coverage (paywalled)
- Media interviews with engineering leaders

## Signals to capture per company
- Tenancy and isolation model
- Sharding/partitioning strategy
- Search/vector adoption inside product
- Reliability targets and incidents
- Cost optimization and storage tiering
- Migration or replatforming decisions
